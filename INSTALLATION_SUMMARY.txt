🎯 QUICK START (3 STEPS):

  1. CLONE (On 10.4.3.208):
     $ cd /opt
     $ git clone http://gitlab.casmart.internal/arquitectura/authentik-login-designer
     $ cd authentik-login-designer
     → All code and configs from GitLab

  2. DEPLOY (On 10.4.3.208):
     $ cat > .env <<EOF
       DATABASE_URL=postgresql+asyncpg://designer_user:SECURE_PASSWORD@postgres:5432/authentik_login_designer
       VALKEY_URL=redis://valkey:6379/1
       ADMIN_API_KEY=SECURE_RANDOM_KEY
       CORS_ORIGINS=http://localhost:3000,http://localhost:80,https://identity.casmart.internal
       PUBLIC_API_BASE_URL=https://identity.casmart.internal
       EOF
     $ chmod +x deploy.sh && ./deploy.sh
     → Builds images, starts services, runs migrations

  3. CONFIGURE (Nginx):
     $ sudo cp nginx-gateway.conf /etc/nginx/sites-available/identity.casmart.internal
     $ sudo ln -s /etc/nginx/sites-available/identity.casmart.internal \
       /etc/nginx/sites-enabled/
     $ sudo nginx -t && sudo systemctl reload nginx
     → Gateway ready at https://identity.casmart.internal/

  "Database connection error?"
  → docker exec authentik-login-designer-db pg_isready -U designer_user
  → Check DATABASE_URL in .env

  → sudo nginx -t
  → docker-compose logs frontend
  → Check ports 80/443 are open
